# Лабораторная работа №4. Асинхронный исполнитель задач

## Обзор проекта

Данная работа реализует асинхронную систему обработки задач, которая собирает задачи из нескольких источников, сохраняет их в очереди и передает обработчикам через `AsyncExecutor`.

## Ключевые архитектурные решения

1. **Асинхронный исполнитель**: `AsyncExecutor` управляет запуском обработчиков через `async with` и `async/await`.
2. **Контракты через Protocol**: источники и обработчики описаны через явные контракты.
3. **Контекстные менеджеры**: исполнитель использует асинхронный контекстный менеджер для управления жизненным циклом.
4. **Централизованное логирование**: ошибки и ключевые события фиксируются через единый logging-конфиг.
5. **Расширяемая архитектура**: новые источники и обработчики можно добавлять без изменения ядра системы.
